# 薄荷记账 部署文档（成品版）

> 本文档面向宝塔面板部署。无需数据库，数据以 JSON 文件形式存储在服务器。
> 保存好此文档，换服务器时按步骤重来即可。

---

## 一、项目文件说明

| 文件/文件夹 | 说明 |
|------------|------|
| `demo2.html` | 薄荷记账主页面（唯一入口） |
| `index.html` | 自动跳转到 demo2.html |
| `data-api.php` | 数据接口：读写 data.json，自动备份 |
| `data.json` | 数据文件（部署后自动生成） |
| `backup/` | 自动备份目录（每次保存时生成） |
| `_shared/js/echarts.min.js` | 图表库 |

---

## 二、环境要求

- **PHP 7.4+**（宝塔软件商店一键安装）
- **Nginx**（宝塔默认已装）
- **无需 MySQL / 无需数据库**

---

## 三、宝塔部署步骤

### 步骤1：安装 PHP
1. 宝塔面板 → **软件商店**
2. 搜索安装 `PHP 7.4` 或更高版本
3. （Nginx 一般默认已安装）

### 步骤2：创建站点
1. 宝塔 → **网站** → **添加站点**
2. 填写域名（如 `book.yourdomain.com`）
3. 根目录：`/www/wwwroot/book.yourdomain.com/`
4. **PHP 版本** 选择你刚安装的版本（如 PHP-7.4）
5. **不要勾选创建数据库**（本项目不需要）
6. 点击提交

### 步骤3：上传源码
1. 宝塔 → **文件** → 进入站点根目录
2. 上传以下文件/文件夹：
   ```
   demo2.html
   index.html
   data-api.php
   _shared/
   ```

### 步骤4：配置 Nginx 伪静态
1. 宝塔 → **网站** → 点击站点右侧 **设置**
2. 选择 **伪静态** 标签
3. 选择规则为 `thinkphp`，或手动填入：
   ```nginx
   location / {
       try_files $uri $uri/ /index.html;
   }
   location ~ \.php$ {
       include snippets/fastcgi-php.conf;
       fastcgi_pass unix:/run/php/php7.4-fpm.sock;
   }
   ```
4. 点击保存

### 步骤5：设置目录权限
1. 宝塔 → **文件** → 进入站点根目录
2. 选中所有文件 → 点击 **权限**
3. 设置为 `755`，所有者和用户组都改为 `www`

> 这一步很重要，否则 PHP 无法写入 data.json。

### 步骤6：验证部署
1. 浏览器访问 `http://你的域名`
2. 应该自动跳转到 demo2.html 并显示薄荷记账页面
3. 点击右上角 **🔄 重置** 或直接记一笔，然后刷新页面
4. 如果数据还在，说明 data.json 写入成功

---

## 四、数据备份与恢复

### 自动备份
每次保存数据时，`data-api.php` 会自动在 `backup/` 目录生成一份时间戳备份，只保留最近 20 个。

### 手动备份
访问 `http://你的域名/data-api.php?action=export` 可直接下载 data.json 文件。

### 手动恢复
把备份的 `.json` 文件上传覆盖根目录的 `data.json` 即可。

---

## 五、换服务器迁移指南

1. 新服务器安装宝塔 + PHP 7.4 + Nginx
2. 创建站点（PHP 版本）
3. 上传所有源码文件
4. 如果有旧的 `data.json`，一并上传（数据就回来了）
5. 配置 Nginx 伪静态
6. 设置文件权限 755，所有者 www
7. 访问测试

---

## 六、常见问题

### Q1：访问页面空白
- 检查 `demo2.html` 和 `_shared/js/echarts.min.js` 是否上传完整
- F12 控制台查看报错

### Q2：记账后刷新数据丢失
- 检查 `data-api.php` 是否有写入权限
- 宝塔 → 文件 → 根目录权限设为 755，所有者 www
- 检查 PHP 错误日志

### Q3：图表不显示
- 检查 `_shared/js/echarts.min.js` 是否成功加载
- F12 → Network 面板看是否有 404

### Q4：如何清理所有数据重新开始
- 删除根目录的 `data.json` 文件
- 或者点击页面中的 **🔄 重置演示数据**

---

## 七、技术说明

- 数据存储：`data.json`（服务器文件）+ `localStorage`（浏览器本地缓存）
- 双保险：即使服务器写入失败，数据也会保存在浏览器本地，不会丢失
- 每次保存自动备份旧数据到 `backup/` 目录
- 无需数据库、无需配置、上传即跑
